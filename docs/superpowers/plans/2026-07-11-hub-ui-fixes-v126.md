# Hub UI Fixes v1.2.6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship seven small UI fixes to the Campaign Record module — import footer layout, timeline thumbnail toggle, a unified doctype filter, index-row cleanup with doctype icons, a full index in the record view's left pane, and a drag-free scene picker — as one v1.2.6 release.

**Architecture:** A Foundry VTT v13 module. The Hub is an `ApplicationV2` built from a `HubBase` mixin (`scripts/apps/hub/hub-mixin.mjs`) rendering named PARTS (`header`, `index`, `timeline`, `record`) from Handlebars templates in `templates/hub/`. Record sheets extend `BaseRecordSheet` (`JournalEntryPageHandlebarsSheet`). Pure logic lives in `scripts/logic/` and `scripts/data/` and is unit-tested with vitest; UI behavior is covered by Playwright e2e against a live Foundry install.

**Tech Stack:** JavaScript ESM (`.mjs`), Handlebars templates, Foundry `ApplicationV2`/`DialogV2`, Font Awesome icons, vitest (unit), Playwright (e2e).

## Global Constraints

- Foundry v13 `ApplicationV2` idioms only; action handlers are `static #on…` private methods registered in `DEFAULT_OPTIONS.actions` / `static PARTS`.
- No new npm dependencies. Module code stays ESM `.mjs`.
- Every user-facing string is a localization key present in `lang/en.json`; `tests/i18n-coverage.test.js` fails on a used-but-missing key. Add keys when you add usages.
- Pure logic → vitest (`npx vitest run <file>`). UI/integration → Playwright e2e.
- **e2e runs are gated:** before running any Playwright spec, restarting Foundry, or touching the module symlink, follow the `foundry-e2e` skill (shared-install session lock, unlock afterward). Run a single spec with `npx playwright test tests/e2e/<file>`.
- Commit after each task with a `feat:`/`fix:`/`refactor:` message.
- Final version string is **1.2.6** in `module.json`.
- Icons are Font Awesome classes written in full, e.g. `"fa-solid fa-user"`.

Create `scripts/logic/doctype-filter.mjs`:

```js
import { RECORD_TYPES, recordIcon } from "../constants.mjs";

/**
 * View model for the Index doctype filter. Selected types become removable
 * chips; the rest are offered in the "add" dropdown. Pure — the caller passes
 * a label resolver so this stays testable without Foundry's i18n.
 *
 * @param {Set<string>} selected  active short types
 * @param {(type: string) => string} labelOf  localized label for a short type
 * @returns {{chips: object[], available: object[], hasSelection: boolean}}
 */
export function buildDoctypeFilter(selected, labelOf) {
  const types = [...RECORD_TYPES, "journal"];
  const chips = types
    .filter((t) => selected.has(t))
    .map((t) => ({ type: t, label: labelOf(t), icon: recordIcon(t) }));
  const available = types
    .filter((t) => !selected.has(t))
    .map((t) => ({ type: t, label: labelOf(t), icon: recordIcon(t) }));
  return { chips, available, hasSelection: chips.length > 0 };
}
```

- [ ] **Step 3: Create the scene picker**

Create `scripts/apps/scene-picker.mjs`:

```js
/**
 * Prompt for a Scene the current user can see; resolves to its UUID, or
 * null if cancelled. Exists because core gates dragging Scenes out of the
 * sidebar to GMs, so players need a drag-free way to link scenes to records.
 */
export async function promptSelectScene() {
  const scenes = game.scenes
    .filter((s) => s.visible)
    .sort((a, b) => a.name.localeCompare(b.name));
  if (!scenes.length) {
    ui.notifications.warn(game.i18n.localize("CAMPAIGNRECORD.Warning.NoScenesToLink"));
    return null;
  }
  const options = scenes
    .map((s) => `<option value="${s.uuid}">${foundry.utils.escapeHTML(s.name)}</option>`)
    .join("");
  return foundry.applications.api.DialogV2.prompt({
    window: { title: "CAMPAIGNRECORD.LinkScene" },
    content: `
      <div class="form-group">
        <label>${game.i18n.localize("CAMPAIGNRECORD.SelectScene")}</label>
        <select name="scene" autofocus>${options}</select>
      </div>`,
    ok: {
      label: "CAMPAIGNRECORD.Link",
      callback: (event, button) => button.form.elements.scene.value
    },
    rejectClose: false
  });
}
```

Note: `Scene#visible` is the core getter for "the user can see this scene." If e2e shows players get an empty list for scenes they should see, switch the filter to `game.scenes.filter((s) => s.testUserPermission(game.user, "LIMITED"))` and re-run.
